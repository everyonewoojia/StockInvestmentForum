## UC-001 用户注册场景

**关联用户故事**: US-001

**前置条件**: 用户未登录

**基本流程**:
1. 用户访问论坛首页，点击“登录/注册”按钮
2. 系统显示登录页面
3. 用户切换到“注册”选项卡
4. 用户填写用户名、手机号/邮箱、昵称、密码
5. 用户点击“注册”按钮
6. 系统校验用户名和手机号/邮箱是否唯一
7. 系统使用PBKDF2对密码进行哈希加盐处理
8. 系统在forum_users表创建新用户记录
9. 系统生成JWT Token返回给用户
10. 用户自动登录并跳转到论坛首页

**异常流程**:
- 用户名已存在：提示“用户名已被占用”
- 手机号/邮箱已注册：提示“该手机号/邮箱已注册”
- 必填项为空：提示“请填写完整信息”

**后置条件**: 用户成功注册并登录，获得JWT Token

## UC-002 用户登录场景

**关联用户故事**: US-002

**前置条件**: 用户已注册，未登录

**基本流程**:
1. 用户访问论坛首页，点击“登录”按钮
2. 系统显示登录页面
3. 用户输入账号（用户名/手机号/邮箱）和密码
4. 用户点击“登录”按钮
5. 系统查询用户记录，提取密码哈希和盐值
6. 系统对输入密码进行PBKDF2哈希，与存储值比对
7. 系统检查用户状态（是否被封禁）
8. 系统生成JWT Token
9. 系统返回用户信息和Token
10. 前端存储Token，跳转到首页

**异常流程**:
- 账号不存在：提示“账号或密码错误”
- 密码错误：提示“账号或密码错误”
- 用户被封禁：提示“账号已被封禁，请联系管理员”
- 多设备登录：正常，JWT无设备限制

**后置条件**: 用户成功登录，Token存储在本地

## UC-003 浏览板块场景

## UC-004 发布帖子场景

**关联用户故事**: US-008, US-029

**前置条件**: 用户已登录

**基本流程**:
1. 用户在首页点击“发帖”按钮
2. 系统显示帖子编辑界面
3. 用户选择所属板块和帖子类型
4. 用户填写标题、摘要、正文内容
5. 用户可选关联股票代码、上传图片和附件
6. 用户点击"发布"按钮
7. 系统检查帖子内容是否包含敏感词（forum_sensitive_words）
8. 帖子状态设为PENDING_REVIEW
9. 帖子写入forum_posts表
10. 系统提示"发布成功，等待管理员审核"

**异常流程**:
- 未登录：跳转到登录页面
- 必填项缺失：提示“请填写完整信息”
- 标题过长/过短：提示字数限制

**后置条件**: 帖子进入待审核队列

## UC-005 帖子审核场景

**关联用户故事**: US-024

**前置条件**: 用户为管理员或版主

**基本流程**:
1. 管理员进入管理后台 → 内容审核页面
2. 系统分页展示所有待审核帖子
3. 管理员点击帖子可查看完整内容
4. 管理员选择“通过”或“驳回”
5. 若驳回，需填写驳回原因
6. 系统更新帖子状态为PUBLISHED或REJECTED
7. 系统记录审计日志
8. 若通过，系统记录发布时间（publishedAt）
9. 列表更新，移除已处理的帖子

**后置条件**: 帖子状态变更，审计日志记录操作

## UC-006 浏览帖子详情与评论场景

**关联用户故事**: US-009, US-010, US-011

**前置条件**: 帖子状态为已发布

**基本流程**:
1. 用户在帖子列表中点击感兴趣的帖子
2. 系统加载帖子完整内容（标题、正文、附件、楼主信息）
3. 系统加载该帖子的所有评论（含嵌套回复）
4. 系统自动增加该帖子的浏览量
5. 用户阅读帖子和评论
6. 用户若想评论，在评论框输入内容
7. 用户点击“发表评论”
8. 系统检查评论内容是否含敏感词
9. 评论保存至 forum_comments表
10. 帖子commentCount增加
11. 前端刷新评论列表

**异常流程**:
- 未登录评论：跳转到登录页面
- 评论含敏感词：正常发布（系统后台标记为待审核），或直接拒绝

**后置条件**: 浏览量增加，可能产生新评论记录

## UC-007 互动操作场景

**关联用户故事**: US-012

**前置条件**: 用户已登录

**基本流程**:
1. 用户浏览帖子或评论
2. 用户点击点赞/收藏/分享按钮
3. 系统检查forum_interactions中是否存在相同记录
4. 如果不存在：创建新记录，`active = true`，对应计数器 +1
5. 如果存在且 `active = true`：将 `active` 设为 false，计数器 -1（取消操作）
6. 如果存在且 `active = false`：将 `active` 设为 true，计数器 +1（重新操作）
7. 前端更新按钮状态和计数器

**后置条件**: 互动状态切换，计数器更新

## UC-008 关注与动态流场景

**关联用户故事**: US-014, US-015

**前置条件**: 用户已登录

**基本流程**:
1. 用户浏览其他用户的帖子或资料
2. 用户点击"关注"按钮
3. 系统在forum_follows表创建关注记录（默认 `starred = false`）
4. 按钮变为"已关注"
5. 用户切换到"关注动态"页面
6. 系统查询该用户关注的所有用户
7. 系统查询这些用户的已发布帖子，按时间倒序排列
8. 用户可翻页浏览

**异常流程**:
- 关注自己：可关注但无实际意义
- 重复关注：系统幂等处理

**后置条件**: 关注关系建立，动态流更新

## UC-009 搜索场景

**关联用户故事**: US-021, US-022

**前置条件**: 无

**基本流程**:
1. 用户在搜索框输入关键词
2. 用户可筛选搜索类型（全部/帖子/用户/股票）
3. 用户点击"搜索"或按回车
4. 系统在forum_posts、forum_users、forum_stock_symbols中模糊匹配
5. 系统按类型分组展示搜索结果

**扩展流程（搜索建议）**:
1. 用户输入股票代码前缀
2. 系统从forum_stock_symbols表匹配返回建议列表
3. 用户点击某建议跳转到相关帖子

## UC-010 用户认证场景

**关联用户故事**: US-005

**前置条件**: 用户已登录，未提交过同类认证

**基本流程**:
1. 用户进入个人资料页 → 认证管理
2. 用户选择认证类型（基础/实名/专业）
3. 基础认证：系统自动通过，直接获得徽章
4. 实名认证：用户填写真实姓名、身份证号，提交后待人工审核
5. 专业认证：用户填写执业机构、执业编号，上传证明材料，提交后待人工审核
6. 系统创建forum_verifications记录
7. 审核通过后更新用户的verificationLevel和professionalBadge

**后置条件**: 用户认证等级更新，获得对应徽章

## UC-013 群组场景

**关联用户故事**: US-018, US-019

**前置条件**: 用户已登录（创建/加入）；未登录（浏览）

**创建群组**:
1. 用户进入“群组”页面
2. 用户点击“创建群组”
3. 用户填写群名称、描述、可见性、加入策略
4. 系统创建forum_groups记录
5. 创建者自动成为群主（forum_group_members中 role = OWNER）

**加入群组**:
1. 用户浏览群组列表
2. 用户点击感兴趣的群组
3. 系统检查群组的加入策略
4. 公开群组可直接加入
5. 系统创建forum_group_members记录

**后置条件**: 群组/成员关系建立

## UC-017 板块管理场景

**关联用户故事**: US-027

## UC-018 敏感词管理场景

**关联用户故事**: US-028

**前置条件**: 用户为管理员或版主

**基本流程**:
1. 管理员进入管理后台 → 敏感词管理
2. 系统展示敏感词库列表（分页）
3. 管理员可“新增敏感词”：输入词语、选择分类、设置启用状态
4. 新增后，后续发帖/评论包含该词时自动进入审核
5. 管理员可“删除敏感词”

**后置条件**: 敏感词库更新
